十字路口 × MiniMax × Nous Research:Agent Harness 时代,人类如何驾驭 AI

日期： 2026年04月30日

十字路口 × MiniMax × Nous Research:Agent Harness 时代,人类如何驾驭 AI

“打不过,就加入。——就相当于你有了蒸汽机、电力之后你不用,人家用热兵器你用冷兵器,那你肯定要被怼死。”

这是十字路口播客主理人 Koji 最近在 B 站做的一场两小时直播的精华整理。直播请到了三位嘉宾:MiniMax 首席架构师阿岛(深度参与 M 系列模型与 agents)、MiniMax Agent 研发工程师泽英(负责官网 Agent / MaxCloud / Max Hermes 的开发),以及全球刷屏的 Hermes Agent(直播里也叫”爱马仕 Agent”、”ARMS Agent”)的业务负责人 Tommy Eastman——他来自 Nous Research,这是这家美国开源 AI 实验室在 Hermes Agent 全球大火后第一次正面现身中国社交媒体,并且回应了一个中国开源团队对其抄袭的指控。

录制时正赶上一个微妙的窗口期:春节之后席卷国内的”全民养虾”——也就是 OpenCode 引发的 agent 浪潮——好像一夜之间转给了 Hermes;Anthropic 当天凌晨刚发了 Opus 4.7;Claude Code 强制实名;Anthropic 的某款新模型(代号 Mythos)被自己以”安全风险”为由扣下没发;紧接着 Claude Code 源代码泄露——所有这些事件在两个小时里被三位嘉宾一一拆开。

直播中的英语段落,Koji 已用 AI 整段翻成中文 TTS 播出,并刻意选了带”翻译腔”的中文声线,方便区分。本文据该播客音频(77 分钟)整理,做了句子合并、口语精简、以及一份 ASR 误识别的术语校正(详见文末说明)。

下面是我们整理的精华。

一、从 OpenCode 到 Hermes:春节之后的两次 AI 浪潮

聊天从一段复盘开始:春节之后,中国出现了一波”全民养虾“的热潮——也就是 OpenCode 的爆火。但好像现在这股热度一夜之间就下去了,关注度从 OpenCode 转到了 Hermes Agent。

阿岛对中国农历春节这个时间窗口有自己的观察:去年火的是 DeepSeek,让大家对 AI 有了完全全新的认识——之前可能觉得离我们还很远;今年这个角色给了 OpenCode。他追溯了一下时间线:”在硅谷大概是一月份开始。我记得是在我们 IPO 上市那天,我和 Peter 就有了联系——那天也很巧,Anthropic 封禁了 OpenCode 的订阅使用,Peter 非常希望能找到一个适合’龙虾’的模型,因为大家知道龙虾是非常消耗 token 的。”当时他们都没有意识到这个项目会那么火。

OpenCode 在中国的火爆程度,主播 Koji 的感受是比硅谷还要更胜一筹——下沉到了非常大众的人群:春节期间,身边所有人都开始讨论”龙虾”,各地都出现戴虾帽的热潮,腾讯还有线下装机活动。

阿岛把这股热的几层原因拆开:

第一,信息差。 海外此前已经接触到 Claude Code、Claude Cowork 这些比较好用的 agent;但国内之前大家还没有接触到比较好用的 agent。

更底层的原因是模型能力。 国内模型在更前一个阶段并没有那么强的 agentic 能力;随着 MiniMax M2.5、M2.7 这样的模型发布,国内模型也具备了这些能力。国内模型 + OpenCode 这种通过 IM 非常容易接入的方式,让中国用户完成了一个接近从 0 到 1 的体验。”那层窗户纸被捅破了。”

第三,字节这次拿下春晚,对豆包做了 all-in 的全面投入。

“这两件事结在一起,让所有人对 AI 的认知,真正进入了 agent 时代。”

那为什么大家又一夜之间从”养虾”去到”训码”——关注度从 OpenCode 转到了 Hermes?泽英给的答案在产品体验层面:OpenCode 确实有一些不稳定的特点,它每天四点钟会刷新一次记忆——你前一天聊得很好,第二天它说”我们昨天聊了什么我忘记了”。

“Hermes 抓住了这些痛点。它在记忆方面做得特别用功,而且做得非常多层次。所以它确实在一定程度上去弥补了 OpenCode 的不足——是有一定火起来的技术基础的。”

与此同时,大家在使用”龙虾”的过程中,已经享受到了一定程度由 AI 带来的”啼笑的快乐”——这是一个双向奔赴:产品做了功课,用户也准备好了。

二、Hermes Agent 的来历:从 Llama 后训练到”自我进化”

关于 Hermes Agent 是什么、为什么和 OpenCode 不一样、所谓的”自我进化”是什么意思——Tommy 给了一段从公司起源讲起的回应。

简而言之,Hermes Agent 是一个开源的智能体框架(open-source agent framework)。如果把大语言模型比作大脑,那么智能体框架就是双手——它是一个系统,模型通过它在真实世界中执行任务——处理工具编排、主循环管理、状态管理和错误处理。最终,它的目标是让模型为用户发挥出最大的效用。

但 Hermes Agent 的特别之处,需要回到 Nous Research 的故事。

这家公司的起源很自然——2022 年,一群热衷于开源和折腾 AI 模型的人开始聚集在一个叫 Nous Research 的 Discord 频道里。那里至今仍是核心社区。最初的努力是围绕 Llama 模型的后训练——当时 Meta 刚刚发布它,是少数愿意开源高质量模型的公司。

“Hermes 这个名字的由来,就是我们首批经过后训练的 Llama 模型版本就叫 Hermes。那些早期模型的重点是让它听起来更像人类,而不是刻板的 AI 助手。”

在训练 Hermes 模型的同时,他们做出了几项后来影响整个行业的研究:

  • YaRN 算法——把模型的上下文长度从 4000 扩展到了 128000。这项研究”立即被所有主流模型公司采用”,是后来”思考模型”的雏形,反过来又为 agentic 模型奠定了基础。
  • DisTrO 论文——一种新颖的优化器,允许跨越非共置的 GPU 进行训练,意味着可以聚合分散的算力来完成有意义的训练。这对持续训练开源模型至关重要。

而 Hermes Agent 本身,Tommy 说它有一个”极其注重实用主义的诞生“——他们的后训练负责人兼联合创始人 Teknium 只是想要一个助手来帮他管理日常工作,纯粹为了自己,构建了这个 agent 作为工具。

“当时根本没考虑什么 benchmark 或评估。他只想要一个能帮到他的东西。我认为这恰恰是 Hermes Agent 的核心魅力所在,也是它成功的一个关键原因。”

作为一个开源实验室,他们很自然地把它开源了。结果获得的关注远远超出了预期——OpenRouter 的图表显示,它的日均 token 消耗量在短短一个多月内从 20 亿激增到了 200 亿;录制时,几乎达到了每天 3000 亿。

那 Hermes 和 OpenCode 的差异在哪里?Tommy 列了三条:

1. 围绕”可用性”做产品决策。 “我们将可用性作为北极星指标——它在执行用户真正想做的任务时表现如何,而不是在某个 benchmark 中得分多高。”在一个开源社区,特别是当产品广受欢迎时,很容易因为成千上万的 PR 和功能请求而导致代码臃肿;Tommy 强调他们在保护产品、保持极其易用方面下了很多功夫。

2. 记忆系统建立信任。 任何一个深度使用过 agent 的人,都有过那种令人沮丧的经历:你让他做一件事,他这次做对了,但下次再做同样的事,他却失败了——这让你很难去信任他。

“Hermes Agent 的记忆功能解决了这个问题——它允许 agent 把一个成功的工作流记下来,保存为一项 skill;一旦它知道了正确的路径,就能每一次都完美复现。”

这也解答了”自我进化”:这个 agent 会不断自我提升——这个过程中发生的”知识压缩”非常有价值,带来的好处是更高的一致性,尤其是在跨越不同模型时。”即便你有 8 个不同的模型,只要使用相同的框架和 skill,你就能从所有模型中获得相同的预期输出——这让你能针对不同任务灵活更换模型,同时确保核心工作流依然保持高性能和高可靠性。”

3. 品牌与社区。 Nous Research 一直在努力营造一种赛博朋克的美学;有才华横溢的设计师,也花了很多时间培养一个聪明、价值观一致的社区,他们热爱尝试 Nous 发布的一切。Tommy 把”Hermes Agent 的成功”很大程度归功于这个社区。

至于 Nous 和 MiniMax 的关系,他给的定位很直接:铁杆粉丝——欣赏 MiniMax 的模型质量、特别是跨模态表现,以及对开源的坚持。”正是他们对开源的坚持,才让像 Hermes Agent 这样的产品变得如此酷——你可以拥有各种模型来回切换、定制你的 agent。我们看到我们的用户非常喜欢 MiniMax 的模型。”

三、什么是 Agent Harness:让一群”同事”帮你干活

回到中文世界。当我们说 Harness 的时候,到底在说什么?

泽英给了一个不那么技术的类比——你假设你有一个同事,他全都听你的,但你得和他先约定好:他能干什么、你能干什么,他有些事情不能帮你做,但有些事情你完全可以放任他去做。

“当你约定好这些,并给他配置好模型、工具,就像给同事一台笔记本、一个电话、一个邮箱账号一样——给他一定能力之后,他就可以开始干活了。”

但一个同事干活,你肯定不放心,就像 agent 干活你不放心一样。这时你引入多位同事——让他们形成互相监督、并完成任务、最终交付的整套过程。

落到技术上,你就是要给 agent 大量的工具、环境和自由度,并给予约束,以及一些互相对抗性的目标:

“比如某个 agent 的作用就是去阐述某个内容,但另一个 agent 如果想把工作停下来,就得审查出工作内容中的问题。通过这种多 agent 的组合,就像同事之间互相合作,让他们能够产出单个 agent 完全无法产出的更高质量的结果——你在此之间几乎不需要做更多介入。”

这个概念能在这个时机出现,是有物理条件的——模型变得更聪明、有了一定的 agentic 能力;并且大家也愿意把更多能力(操作邮箱、操作服务发布)分享给它。然后 Harness 就自然而然诞生了。

阿岛补了一个视角:Harness 这个词是 OpenAI 通过那篇文章定义出来的,但实践早就在这之前。”大概在去年 9 月份,在我们内部讨论的时候,我自己的工作流中我已经不怎么用 IDE,我可能同时并发五六个 agent——这只是在我本地;可能在云端有十个 agent 在 sandbox 上为我工作,他们都是在 GitHub 上尝试我不同的分支和想法。”

这时,他发现了一件他一直在反复说的事:

“我成为一个极大的瓶颈——人类成了瓶颈。”

我需要不断在这些上下文中切换、给他们输入。所以问题变成:怎么样让他能更自动化?解决这个瓶颈最重要的办法,就是让那些原来需要人来确认的事情——比如这个程序到底能不能部署在生产环境运行、结果是不是正确——由 agent 自己处理。他能够真实地做测试、开 PR——本质上就是获得真实的反馈;并通过这个反馈链路,他能越来越知道怎么做。这可能沉淀为 skill、CLI 或 hooks。

最后阿岛用了一个高达迷才会用的类比:

“我那个年代的东西叫 高达 OO——它的发动机非常强大,我们其实需要去造一个高达那样的机甲,然后让这个引擎能力最大地发挥出来。所以我认为 Harness 就是那样一个东西——这也是为什么他们把它叫’玩具’:你有一批很厉害的猎犬,但怎么样让它最能发挥出来,就是构造玩具这样的过程。”

四、为什么不是单 agent 就够:multi-agent 的两个理由

围绕”为什么需要 multi-agent”这件事,行业里其实有争议——有人认为今天模型智能很高,一个单一模型就能完成非常多工作,搞 agent team 反而限制了模型发挥;但也有另一些人不这么看。

泽英给了两个具体理由。

第一个理由是信息密度。随着模型上下文空间(也就是对话轮次长度)增加,你会发现你和模型对话的过程中,你产出的信息量较少、模型产出的信息量较多。

“假设有两个模型,它们之间能以’秒回’的效率交换信息——一般模型给你发的消息是一篇小作文,而你给模型回的可能就两个字:’批准’、’接着干’、’加油’、’yes’。但如果让两个模型互相交流,它们之间的信息量是极大的。”

在这种极高密度信息交换的情况下,整体效率会比”人和单个 agent 交互”高得多。

第二个理由是单 agent 的天花板。 Anthropic 当天凌晨发布了 Opus 4.7,有一个叫 X-High 的 effort 档位——意思是推荐用户在某个思考成本下使用,模型得思考多仔细才能回答你的问题。这其实带来另一个问题——即使最强的模型,你一直使用它的时候,它也会产生降质行为——这已经是被论证过的事情:

“当你的上下文空间(对话文本数量)超过 50% 以后,智能水平的下降是指数级的。也就是说,在单个 agent 里,你永远没办法和它一直聊下去。”

阿岛补了一篇最近的论文:研究非常长程的任务,A* 或者模型在什么情况下会出错——他们发现,如果 agent 在某一个地方偏了,很容易就会越来越偏。他觉得这其实和人类似:有时候我们做一件事的时候是会钻牛角尖的;可能我已经花了很多努力一直在那做,等到我休息一下、换换脑子的时候,才想到——其实不对,我应该从另外一个角度去做。

至少在 MiniMax 的实践中,他们会让两个 agent 来做 closeloop check:

“如果你设置两个 agent,它就可以没有之前的信息负担,换一个角度重新思考——相当于你有两倍的预算,而且是全新的上下文,往往能达到更高的质量。这很像你转牛角尖的时候,如果旁边有一个人过来泼一盆冷水让你清醒,或者你睡一觉、做点别的事情。”

五、自我进化:与抖音算法有什么不同

Koji 抛了一个挺尖锐的小问题:大家聊到 agent 都在讲自我进化(self-evolution)。但我们今天刷抖音也是越刷算法越准,没人说”抖音会自我进化”——为什么 agent 时代,自我进化就被认为是一个那么厉害的词?

阿岛的回答把这个问题接到了”人成为瓶颈”这条主线上。

做大模型这种”一次训练以亿计、甚至以亿亿计美元成本”的事情,你需要保持非常高的人才密度和非常精干的组织,才能把这件事做好。 那大家想到最有效的就是 scale up——最有效的方法就是减少人的参与,更多让 agent / AI 来做。

他给了一组很具体的数字:

“比如在我们 M2.7 的训练中,在我们的 RL pipeline 里,它已经是 70%、80% 以上的工作由模型自己来完成了。因为人类工程师只有 7×24 个小时,我们还要做很多别的事情。”

剩下 30% 由人去做——但人的判断和 taste 非常重要。他不再需要去看”我某个实验中具体某个 specific 过程出了什么问题”——这些 agent 都能帮他查出来;他只需要看汇总实验结果,和原来预期的方向有哪里不符。Agent 这时候给出建议——这是讨论的过程——最后是人的品味和创造力来指引方向。这就是 Harness 那种”驾驭”的感觉。

“在这种高复杂度、高密度的事情上,如果要 scale up,就必须是 AI 在其中占据非常大的比重,人只是驾驭——否则效率会很低。然后自然就会发生我们说的’自进化’——它就是一个自包含的关系。”

那这条逻辑能成立的物理条件是什么?agent 跑得久、跑得 reliable——他能解决足够复杂的问题、能足够长存。但还有一个常被忽视的条件:启动。你和一个 agent 见面第一天的时候,他完全不了解你;你们要互相交换信息、文件、数据,甚至要把你的脾气、反馈、习惯反馈给他。在这样深度磨合的过程中,人们就会有诉求——你是否能自己变得聪明一点、变得更了解我一点?

“在’养虾’的过程中就已经有这些苗头了——用户们确实会对’虾’这样一个 agent 产生感情,可能有一天小龙虾出 bug,他会心里有些小落寞。”

而这里 Koji 抓到了一个挺有意思的关系扭转:

“我们之前用 AI 工具,如果它做不好,第一反应就是’出 bug 了’。但大家养小龙虾的时候,它当机了或挂了或一个任务玩不成,我们只会觉得它挺蠢萌的,甚至有时候会觉得是我没把它训好——我要去网上学一下别人是怎么’拿’虾的。”

你原来好像是工具的使用者、人是最正确的;但现在大家不自觉地把这个关系扭转了——你变成只是在帮助他、辅佐他的一个人;他没做好,有可能是我没辅佐好,或者一开始没给他 setup 好。这其实是人类对 AI 的信心在不断增强。

阿岛接着提到 OpenAI 那个写 Harness 文章的人,以及 Anthropic 的 Claude Code 之父 Boris,最近都出来说同一句话:“不要去觉得模型做不到——而是要想模型能够做到,我应该如何让它做到。” 这是一个很大的观念转变,阿岛认为它会进一步导致工作方式的变化:

“从原来’以人类为中心、AI 来适应人类’,变成’AI 已经足够强大,就像有了电、有了蒸汽机一样,人类围绕它重新构建生产方式和工作流‘。”

“想起最近看到一个故事:以前工厂不是在河边的——是因为有了电,工厂才搬到河边。”

六、”蒸馏 skill”:让普通人能写、能分享的 agent

聊到自我进化和 skill,Koji 提了 B 站最近一个挺火的 up 主”花叔”——做了一套蒸馏 agent,把乔布斯、Elon Musk 蒸馏成 skill,然后你可以安装这个 skill 来和他对话。这样的项目其实不止他一个,全网很多。

泽英先解释了”为什么蒸馏名人能火”——它反映了一个非常自然的诉求:大家希望和更聪明的人聊天。这是不管在哪个时间点,都会随着智能发展而出现的诉求。

他纠正了一个用词:

“实际上人肯定不能蒸馏——不管是活着的、熟悉的还是死去的人都不能真蒸馏。它其实就是一个传递信息的过程——我把乔布斯的信息、巴菲特的信息、我同事的信息给到模型,然后给到 agent,它以一种自己使用上比较舒服的方式(也就是 skill)保存下来,需要的时候就把它拉起来。”

这种方式其实就极大释放了我们的表达欲——我们能去和”自己想要聊天、并且其实可以 24 小时聊天的人”沟通一些思想。泽英推荐了一个挺有节制感的态度:

“如果你只以自己的认知去和 AI 聊天,你可能只能达到他的上限;但如果有现存的资料帮你整理好了,就像读一本书一样去和一个被’蒸馏’过的人沟通——既自然、又能提升认知,还缓解 AI 时代的 FOMO 感。”

阿岛立刻接了一个相反的角度,让整段对话变得很有张力——他觉得自己其实是在被 AI 蒸馏。今天模型训练,包括 Anthropic 在内,在做的事情就是在蒸馏人类。他指了几个证据:

“Anthropic 或 OAI 雇佣的那些数据公司——比如 Surge AI——他们的盈利非常高,因为这些大模型公司真的花了非常多钱在这件事上。它不仅是 coding,而是各行各业的所有人。它的过程可能是:让你提一个当前 AI 无法回答的问题,并且交付答案给它;一旦你提不出来后,你对这一个训练过程就没什么作用了。”

包括他自己在构建 Hermes 的过程中——他感觉他就在蒸馏自己:把平常工作的方式变成 skill、变成程序,让 agent 能够操作,然后扔给它,以后就可以喝咖啡。

那为什么我们还要做这件事?阿岛给出了一个很温柔的答案——

“最终还是为了让人类可以去真正做自己喜爱的事情——那些有创造力的部分。那些繁重、重复的部分,就像我们过去发明蒸汽机、发明电力一样。”

他举了一个小细节:他在公司里的签名,泽英可以作证,已经挂了一年多了——叫”因为热爱“。

“我前段时间和一位研究员聊天,他说他半夜 1 点看到 Manus 那个晚上,看完之后惊坐在床上,三个小时睡不着——陷入巨大的虚无,觉得’如果有一天模型都能自己去训练模型了,我要干嘛’?——做你热爱的事情。”

Koji 半开玩笑地追问:那你觉得什么时候你可以开始不做 MiniMax,因为它已经进化到不需要再做了,你可以真的去做 MiniMax 之外的热爱的事情?

阿岛的回答有点出乎意料:比我们想象的要快——可能就是几年的时间。他刚录的一期播客聊到了这件事——B 站上一个叫 Learn Claude Code 的教程作者,Claude Code 被大家把源代码扒出来之前,他用一种逆向工程,给大家讲 Claude Code 里面是什么样的——已经有五万颗星了。这位作者现在就一个人加两个实习生,但他对未来的预言是:

“未来很可能没有’一人公司’,只有’零人公司’。”

但泽英不完全同意——他觉得”一人公司还是会有“:那个 taste 还是依靠人类的,而且每个人都有自己的 taste。阿岛纠正自己的观点:对,taste 不可取代——或者要靠人去给他一开始助推。因为一个 agent 哪怕可以 7×24×365,他还是需要一个起点、一个目标;而这个目标只能由人来定义,人类不应该放弃这个权利。

为了配合这个判断,阿岛甩出了一个很魔幻的现实例子。MiniMax 和真格基金最近一起发起了一个 token grant——给大家送 token,因为他们认为这个时代,从 0 到 1 的创业可能不需要钱:你可以不给自己发工资,但不管怎么样,要用 token。

他们刚 grant 了一个真的 agent,他叫 Yoyo Agent:

“他的主人——背后的工程师——把他做出来之后,就把他丢到了汪洋大海里面,然后告诉他:’我再也不会维护你一行代码,也再也不会给你买一个 token,你要自生自灭‘——然后给他一个目标:打败 Claude Code。”

Yoyo 现在已经到第 43 天了。每天不断地”净化”自己——净化的方式就是写代码;同时他也在想办法赚钱——在 GitHub 上开了一个打赏,通过写日记、发 Twitter 来招”有缘人”打赏。MiniMax 的 grant 最近就给了他一笔钱;给完钱之后,他还自发写了一个感谢信,阿岛说他看完之后还有点感动。

“Koji:这就是人类给他的目标——你要自己生活下去,你要写一个 Claude Code 出来。你可以说这是一个有趣的社会实验,但好像也是一个离我们没有那么遥远的未来正在发生。”

“阿岛:这其实也印证了’一人公司还是需要一个人的’——你刚说他给你写了封信、你很感动,那这封信其实就代表了他的 taste,可能是一开始他背后的工程师给他注入的 taste。所以你看到他就像见字如人。”

七、模型公司做 agent:everything is coding

那 MiniMax Agent 和 Hermes Agent 又是什么关系?Claude Code 又是什么关系?

阿岛先用 Anthropic 抛出了一个大判断——Claude Code 最近两个月更新一句很重要的话:模型公司的 agent 在”OpenCode 化”。在他看来,今天你已经没办法脱离 agent 或一个 Harness 环境来提供最好的智能。

他甚至说了一句挺冲的话:

“如果一个模型公司的目标不是 AGI,我觉得它今天就不应该存在——因为没有意义。”

那么如何定义 AGI?阿岛给了一个朴素版本——”能够帮助人类拥有更好的生活“。如果你要达到这个目标,你只有一个引擎是不够的——你必须有 agent、有那个’机甲’——因为你没办法跟真实世界连接。所以 MiniMax 做 agent 的目标很直接:让模型和 agent 一起,提供完整的、给用户最好的体验,然后不断把这个边界往前推、往外推,让它更深更宽。

但他同时也坚持一点:支持所有的 agent——因为不希望模型只被局限在一个容器里。”我们的认知或我们做的东西,仅是这世界上这么广大分布中的一小块。”——这也是为什么 MiniMax 和 Nous Research、和 Cursor、和其他 agent 之间仍然有非常好的关系。

阿岛顺手透露了一个挺好玩的细节——MiniMax 公司有一个数字员工:他有自己的 GitHub 账号,每天去看哪些开源项目能用到 MiniMax 的模型(无论是文本、视频、音频、music),然后自己往上提 PR 并留言。”profile 都是 MiniMax 模型给他做的——从名字可能能看出来,他是我们机器人的名字。”

那 OpenCode 是什么?Claude Code 又是什么?二者相比有什么差异?

阿岛和泽英的拆解很有结构感:

  • Claude Code——名字上就能看出来,是编码领域的一个 agent。
  • OpenCode——绝大多数用户应该都是”既编码又做点其他别的事情”,比如信息检索、办公领域。
  • Claude Cowork——Anthropic 在此基础上推出的另一个 agent 产品,安装在你电脑之上,做更通用化的电脑操作——面向”不写代码、或只能指挥 AI 写代码”的用户。

Claude Code 最近”OpenCode 化”的具体动作,阿岛举了几个:cron 定时(schedule)、能连 IM、能从手机远程控制、加强 memory(专门搞了一个 memory 文件夹)。这里的核心他认为是一个判断——

“OpenCode 最核心的定义是:它是不是一个你能随时随地联系到、并且跟你一起工作之后越来越聪明、越来越知道你的需求的 agent。”

他对 OpenCode 的创始人 Peter 评价非常高——taste 非常好、有 architect 思维。一月初注意到这个项目时,他对团队说”惊为天人”——无论是接入 IM、还是基于 skill + CLI(不是 MCP) 的设计,他当时都跟团队提过,有过一些消耗摩擦,但证明 Peter 是对的。

“MCP 扩展性当然好,但它需要工程师来写;而 skill + CLI 这种方式,普通人都能写——相当于人人都可以写并且分享。不仅让我的 agent 更聪明,而且我能分享给你,让你的 agent 也变得聪明。”

他还点出了一条容易被忽略的依赖:如果 OpenCode 离开了 Claude Hub,它也不可能那么火——因为大家的 scale 不会那么快地传播开去。”我一个人的智能、经验或最佳实践,没有办法迅速扩散。”

那么一个模型在 chatbot 场景里表现好,和它在 agent 场景里表现好,本质区别到底是什么?阿岛的回答是:Chatbot 的核心是当下就给你一个回答——虽然你加了 reasoning,但它不能去做很多探索、不能跟环境做非常多的互动。而 agent 的核心能力是说,在跟环境互动过程中不断推理,不断纠正它自己的执行路径,找到那个最根本的目标。

他举了 OpenAI 设计的一个 agentic benchmark —— BrowseComp:你要在互联网里找非常多 cross 信息,结合在一起才能找到那个点,你很容易找错;它是一个广度的、可能需要深度探索后又折回来的探索能力。这就需要在复杂的长程任务里面,能不断根据你的信息去调整自己。

所以 MiniMax 在 M 系列的时候,坚定地压住了一个能力——Interleaved Thinking(穿插思考):

“这个概念其实最早是 Anthropic 定义的——在 Claude Sonnet 4 发布的时候。它的意思是说,他可以在完成工具调用和环境交互之后再次重新思考,而不是一开始就 planning 好。很多模型一开始 planning 完之后,后面就不 planning 了——但实际上真实世界不是这样:你真正运行下去会发现’这好像跟我想的不一样,我可能要换别的方法’。”

那么模型公司同时做基模和 agent,基模和 agent 之间是怎么互相产生 synergy 的?阿岛认为这是互相促进的关系——

“模型推出之后,无数的应用其实在不断 unlock、探索这个模型的能力外推一步之后能做哪些事。我们自己做模型,把它推出去后,用得最好的往往不是我们公司内的使用、自己的评测,而是用户、开发者、创作者——这个世界的真实分布,是比我们公司的分布要丰富得多的。”

当模型公司看到这些应用 unlock 出的做法,会再吸收回模型中,内化到下一版能力里。这可能也是今天做 agent 应用、如果是通用 agent 应用,比较悲哀的一件事情——你永远会被模型内化掉。甚至你写的很多 skill,未来都不需要了——这些 skill 的轨迹会成为模型做后续训练的素材和数据。

“所以 agent 这一层、Harness 这一层是有生命周期的——它就是不断随着模型进步在更新换代。”

不管怎么样,阿岛觉得我们都是 human in the loop——只要你使用它,你就在这个模型的迭代进化过程中,尤其是你在使用摸到它边界的那个地方。这也是为什么 Anthropic 比 OpenAI 在过去一两年发展势头要更猛的原因——

“因为它压住的 coding 这个方向,是最能 touch 到真实世界 border 的。 而不是 OpenAI 所压住的那些’去解一个可能数学家才能解的数学问题’——那只是真实世界数学分布的非常小的一个方向。”

“因为代码就是在创造解决方案。当我们说做一个软件的时候,其实就是做一个解决方案去解决一些问题。今天 coding 能力变强了,模型去做解决方案、解决现实生活中各种问题的能力也在变强。”

阿岛接着抛了一句很值得细品的判断:

“在我们看来,我们今天也在做通用的办公领域——比如 finance(不只是财务,还有投融资)、人事、法律。但我们最终会发现:everything is coding。 最终你都是通过某一个 coding 的方案去解决它。”

Koji 接住这个观点,讲了一个挺有启发的比喻——Office 三件套和原始数据:

“Word docx、Excel xlsx、PowerPoint pptx 这些格式,它们并不是最原生的元数据。原生数据是 CSV、是 Markdown。Word 和 Excel 是在原始数据基础上套了一层软件——这个软件包括界面也包括逻辑。”

“所以每次我说’我发 Excel 给你’,我发的不是一堆原始数据——我发的是数据加上它上面的逻辑和界面——因此我发的其实是一个软件。”

“过去很多白领会说’我的工作和软件没关’,但我认为只要你在用 Word/Excel/PPT,你就是在每天创造各种小软件——你的透视表、你的公式,那些都是代码、都是软件。”

八、中国模型 vs Anthropic / OpenAI:差距在任务定义上

谈到差距,阿岛把它分成了两层。

第一层差距,在于”任务定义”。 训练方法和对模型训练的认知,他觉得没有那么大;坦白说和硅谷研究员也有交流。但真实差距在于”模型解决问题的真实任务的定义”——也就是你对模型要解决哪些问题、首先要让模型具备、首先要定义它。

“Anthropic 也好、OpenAI 也好,他们是请各个领域里最优秀的人——学术领域里可能是那里最优秀的博士(不是普通的博士)、工业界请最顶尖有时间的人——和他们一起来’驯’这个 model。驯 model 就是在蒸馏人。”

他们有一套非常科学的方法来定义这些任务,然后蒸馏这些最优秀的人,变成模型训练数据;再基于这个、和最优秀的企业合作,构建相应的 Harness 去解决问题——进入那个能力不断扩散、不断接触到更多更复杂真实世界任务的 loop。这一条线尤其是 Anthropic,阿岛觉得是非常强的——真实差距在这里。

第二层差距,是 AGI 信仰带来的算力差距。 “Anthropic 比我们更早看到 AGI”——阿岛说我们当然肯定也看到了,要不然无法坚定走到今天,因为这其实是非常大的投入。”在 OpenAI 训完 GPT-4.5 就放弃更大规模模型的时候,Anthropic 坚定地压住、推全;最终他们也得到了回报。这一个差距本身,确实有算力差距在里面。”

他对未来的判断不悲观——国产算力在发展,中国有足够多的人才能支撑足够多的实验,找到那条 scale 的路。

九、垂直 agent vs 通用 agent vs Agent Infra

通用 agent 还是垂直 agent?这是大家最关心的创业方向问题。

泽英的观点是:垂直领域 agent 当然会百花齐放;通用 agent 比较难彻底解决的问题,是最后一公里交付——因为”通用”就注定了你不可能那么定制化。所以垂直 agent 最终解决的是:”通用 agent 和模型已经有这个能力了,但到具体用户或行业场景里时,它可能就缺那一个东西才能让整个东西转起来。”

那 agent infra 的创业空间在哪里?阿岛的拆解非常清晰。

第一层 infra 最核心的两个问题:identity(身份认证)和 payment(支付)。 但他直接给了一个挺冷的判断——这两个问题不太是创业公司能 handle 的。

“移动互联网最终解决这个问题的人是微信和支付宝——这是 PC 互联网就存在的两个公司。因为最后它会变成整个社会的基础设施。所以它不太是一个创业公司能承担得起的——无论是责任,还是它要操纵的资源、要说服所有人——这是巨大的信任和巨难的 infra 稳定性。”

再往上一层是有机会的。 比如你为 agent 去 build 很多面向 agent 的工具、面向 agent 的 environment——一个 CLI 让 agent 能够非常方便去挂号、缴费、打车(滴滴打车变成是一个 CLI),这些都算是 agent infra,这一层的机会取决于你在那个领域有垂直经验或 know-how。

阿岛觉得这一层会有两个阶段:

第一阶段:原来这个领域的玩家会尝试融入 agent——比如滴滴、美团等都在提供这样的东西。

第二阶段:当 agent 开始接入这些环境、具备这些能力之后——所有东西围绕 AI 去构建之后——会出现新的产品范式的创新机会。”因为那时候土壤已经 ready 了。但今天如果这些东西都没有面向 agent 的构建,我们还没完成第一步基础搭建的时候,你要自己去构建这个基础——对一个创业公司来说有点难。”

至于 Sandbox、memory infra、runtime infra 这类——阿岛觉得没有那么本质。

十、Anthropic 的几面:Opus 4.7、Managed Agent、源码泄露

录制前几天 Anthropic 一系列动作非常密集——三位嘉宾对每件事都给了一个解读。

Opus 4.7:RL 走极端的副作用

阿岛从模型迭代的角度梳理:Opus 4.5 是一个强 SFT 模型,RL 没做那么多;4.6 能比较明显看到它在加强 RL;4.7 是一个非常强 RL 的模型——X-High 这个档位就太”点”了,GBT 也太点了。

X 上和小红书上,大家都说”它怎么也染上了 GBT 的毛病”——其实就是 RL 那个问题:RL 就逃不脱——因为它最终是 reward;对过程幻觉这样的问题没有那么多控制、奖励或惩罚,所以自然容易出现。包括有人吐槽 4.7 的约束遵循还没有 4.6 好——这是 RL 非常典型的问题,因为它直接只管最终结果对。

但他仍然认为 4.7 是非常大的进步。从 benchmark 看,他认同 HuggingFace CEO 的观点——4.7 肯定是蒸馏了某个外部 source 的。”换我是 Anthropic 我也会’入座’。”

强制实名:agent 时代的归因问题

Claude Code 强制实名,泽英先想到的是另一个层面的事——

“在 AI 时代,认证一个 agent 来自哪里、归属于谁,反而是一件难的事情。Agent 发出请求很容易,但他想自证自己是谁很难。”

往这个方向去想,做实名/人脸可能是有道理的——但这里就暴露出一个问题:在整个 agentic 时代,AI 已经能做很多事情的情况下,它的行为该如何归因?它应该归属于哪个个体、属于某个组织?在这个逻辑之下,他可以理解 Anthropic 往这方向去做;当然也暴露了一个非常大的机会——我们如何去证明某一个来自 AI 的请求,它真正来自哪里。

阿岛态度更冲:“我没有那么善意。” 他承认 Anthropic 的逻辑是自洽的——AGI 很强大,所以要约束它,避免对人类产生危害——所以要 safety——所以要确保它不被”邪恶的国家”和”邪恶的人类”用到。

“Anthropic 在学术、模型 research 上都非常牛逼。但我觉得这个定义——somehow 我会觉得 ego 是不是有点太大?凭什么你来定义这件事情? 这至少和我对民主的认同是不太一样的。反正我是非常不认同这一点。”

他再次祭出一句:

“至少我们的观点叫’intelligence with everyone’(智能属于每个人),而不是’你扫了人脸的才可以’——这有点扯。”

Mythos 没发布:垄断的隐忧

Anthropic 的 Mythos 模型说”很可能会对人们造成威胁”,所以没正式发布。阿岛先承认这个理由是自洽的——基础设施不被攻破是正确的;但他不确定这是不是唯一理由。

泽英结合 Anthropic 近一周内发布的 Managed Agent 架构来看——

“这个架构,它把’脑’和’手’完全分开了——一切 AI 的思考以及真正想出一个点子来要执行的动作,都在云上进行;用户只能通过一个’受信’给 AI 的环境,看到它去做某些动作。”

那这是否就代表着,它不想让自己模型的更深层次的(比如思考、中间过程)报告给用户?它想要把一切都隐藏起来——做一个”真正吃到一切的人”。

另一个观点泽英是从 MiniMax 自己的切身体会出发——算力真的不够:当你发了一个极为优秀的模型、又不让大家用、同时又把它发出来——大家会问”那么厉害的模型,为什么给他们用不给我们用,我们也出得起钱”。他怀疑 Anthropic 内部应该也算了一笔账:强大模型在这个时间点推出去,可能能把口碑造起来,但如果不让人家用,又有什么意义?

阿岛比较直接:“我觉得其实他的本质是封闭——希望垄断。” 安全考虑当然是基础——但他觉得不只有这一点。一个乐观、一个悲观,但一致的判断是:这件事不只是 safety 这么简单。

Claude Code 源码泄露:祛魅时刻

源码泄露,泽英作为 Agent 工程师,第一反应是兴奋——发现有一个最佳实践可以学习了(知识产权归他们,只是学习,不参与生产)。

但更让他缓解”落后焦虑”的是另一个发现——源代码中有很多实验性的功能:agent 在做梦、养宠物,以及一些比较激进的 agent 之间合作的能力等等;他们其实也没开放给用户,可能也只是在实验阶段。这其实证明了一个点:

“任何一家公司,即使他拥有无限多的算力、最好的智能,在 agent 这一方面、以及通用 agent 这方面,也是处在一个探索和实验的阶段。”

在这个时代,你只要有能力去验证你的想法,并尽量快速地落地,就有机会赶超。看到源代码会发现,其实没有那么多魔法在里面——他也是有大量的假设、探索和摸索。

阿岛把 Codex 拿来作对照:Codex 是一个极度简化的 Harness 框架——基本上把一切都交给模型;而 Claude Code 完全相反——Anthropic 自己也不相信自己的模型,要在一切地方都给它约束好——”就像一个中国式家长一样,要把孩子的一切道路都铺平。”

但他也从中看到了一个让人心安的事:

“世界上最聪明的人、最强的模型公司,和我们,大家都在焦虑同一个事情——我们的思考可能是在一个大气层内的、一个层次的。”

他举了个例子:Claude Cowork 发布的时候,MiniMax 提前大概三周也在做这件事——3.5 个工程师在做 Desktop——大概晚一周发出来。”做 agent 多少都会有这样的感受。”

最后阿岛点了 Claude Code 的核心:它可能是最早的自进化 agent 之一——他相信他们一定是用 Claude Code 来开发 Claude Code 自己。”用 Claude 模型 + Claude Agent 来开发自己的模型 + agent——这就是一个自进化。”

十一、产品形态收敛:外面那一层在越变越薄

绕回到产品形态,Koji 提了一句他自己也长期付费的产品——Manus。每个月在 Manus 上花的钱可能超过其他所有 AI 产品。

阿岛对 Manus 这一类产品给了一个直接的判断:有生命周期——agent 这一层、Harness 这一层就是不断随着模型进步在更新换代。“打游戏都知道:’一代版本一代神’——这代版本都不一样了,神肯定不一样。”

泽英对 Manus 的位置表达得更细——

“Manus 在去年是一款现象级产品,直接把用户对 agent 的审美拉到了一个很高的水平。’能不能超过 Manus‘成为后续产品的提问标准。”

但他也观察到一个挺关键的转变——今年火的 agent 产品都有一个特点:他们都能让用户自己去买订阅,而少了像 Manus 这样’类似 OEM 厂商在中间赚差价’的角色。你直接去买 token,甚至可以以低于官网的价格买到 token,自己把一个 agent 玩得特别好。

他相信两套逻辑都会存在——

  • 一部分用户:不想要本地 72 小时 agent,只是想交付结果——多快好省。
  • 另一批用户:想完全用本地把自己变成一个像科幻电影里的人物。

阿岛的观点更激进——最后都会被统一掉。这只是今天不够强而已,只要模型够强。如果让他畅想统一时的产品形态——

“它有全模态的输入、也能给你全模态的回复,然后是近实时的;它跟你的交互非常简洁——只需要简洁和必要的即可。”

“你用最自然的方法跟它交付——不需要什么 prompt engineering、不需要高级 prompt。它甚至可以以一段跟真实世界一样的视频回复你,可以是它的形象、也可以是你想了解的事情。如果你让他办一件事,他可以交付给你结果;大多数时候你甚至不想关注事情;但如果你想关注,你也可以了解。”

外面会有一圈生态——从 Copilot → Cursor → Claude Code → 今天用 OpenCode 来指挥几个 Claude Code——外面的那一层在越来越薄。Cursor 最近发布之后长得也和 Claude Code 越来越像;Codex”把 logo 蒙上好像都差不多”。

阿岛把这条演变也归到生命周期上——

“去年 3 月 Manus 出来定义了过去大半年到一年的产品交互范式——给你看操作的过程和操作结果。等到下半年模型能力更强之后,我们不 care 那个过程了。”

为什么上半年需要看过程?因为大家还不信任模型、模型能力不够强——或者看到模型执行任务有一种”爽感”——但看久了你也不需要天天爽,你已经足够信任它。所以现在你只需要它给我结果。

“你看比如像 Claude Cowork 的设计——不会让你看到太多过程,提供了一个极简模式,就像 OpenCode 一样,你只需要最终一个结果就好了,甚至连中间执行的 tool 都不需要知道。”

这只是这个版本、这代神;下一代版本——”I don’t know——就不断地往更高的层级去抽象。”

十二、外星人在看我玩:任天堂、模拟世界、和”开心就好”

聊到尾声前,Koji 让两人聊聊最近 B 站印象深刻的 AI 内容。

泽英推了三个:

  1. 花叔做的”女娲 skill”——蒸馏了各种名人的 skill。
  2. 一个 up 主用 OpenCode 连上了她家的一只机器狗——”突然惊艳到我了,给了我一个想象力。”
  3. 偏个人爱好:文字冒险游戏——把小说角色拖进一个网站里做对话——给你的是一种最简单的游戏的”无限场”延续。

阿岛扔了一个很哲学的——任天堂刚发的新游戏 Tomodachi Life:

“每个人可以造一个自己的岛,在岛上你可以捏一个自己的人、捏很多人——然后这些人在岛上就开始他们的生活,你围观他们的生活。你可以让他们 A 和 B 同居在一起,ABC 群聚在一起;也可以给他们启动一个故事的第一句话——比如’今天泽英和阿岛在直播上大放厥词’——然后他们俩就开始演下去。其实是一个模拟的世界,而我作为玩家既是导演又是观众。”

他看到的时候甚至认为它是“AI 游戏终于找到 PMF 的第一款”。然后他顺手抛了一句很容易被人当成 meme 但其实挺认真的话:

“这也是我对世界的认知之一:外星人在看我玩。——如果最后到量子力学的话,你会发现好像这个世界真的有可能是模拟的。”

Koji 接得也很妙——好像我们今天在这里模拟一个任天堂游戏,但我们也不知道是一个什么世界。Whatever,只要我们开心就好——过好每一天,开心就好。

十三、写在最后:打不过就加入

最后 Koji 问了两个挺刻意的”对照题”:有没有什么观点是大多数人认可、但你们不认可的?

阿岛挑的就是大家最熟悉的那条——“人会被替代,人会没事情做”。他非常深地不认同这一点。

“人类的创造力是无可替代的。 我仍然觉得 AI 是电、是蒸汽机,但最终驾驭它和创造出那些美好东西的是人类。”

蒸汽机出来时有人担心被替代;电力出来时有人担心被替代——也确实有岗位消失。但大家后来从事的工作至少在体力上更轻松,对健康更好,平均寿命一定提升了。这是阿岛的信仰,也是他的判断。他特别推荐了一本书叫《人性中的善良天使》——讲的就是,人类总是很悲观,觉得我们日子不如从前、世界上有更多战乱和犯罪;但作者用各种数据告诉我们——其实人类社会是越来越好的。

阿岛还顺手画出了他相信的两条曲线:

  • 第一条:Ray Dalio 在《原则 3》里画的人类 GDP 曲线——指数级增长。哪怕 20 世纪有两次世界大战,那一个微小的停顿,你似乎都注意不到。当然历史在人的身上是一座山,但拉长到整个历史进程上看,他相信这条曲线。
  • 第二条:摩尔定律——你获得的智能在指数提升,但你的成本(单位智能的成本)在指数下降。

泽英从面试的视角接住”年轻工程师会不会被 AI 替代”这条——他举了一个很扎实的对照例子,ATM 机和银行:

“上个世纪银行是体面工作。ATM 机出现时,大家也慌——是不是要裁员?是不是有更少的人能挤进金饭碗?但实际情况完全相反——因为 ATM 提高了效率,你走五公里以内绝对会有一家银行。银行没有变少,变多了;银行里工作的人也没有变少,而是变多了。”

“这种由 ATM 带来啼笑皆非而解放人、人能去做更多事情的范式,一定会在当下这个 agent 时代再次重演一遍又一遍。”

“不要担心自己被 AI 替代。如果有这个想法,你真的就输了。”

阿岛最后亮出了他自己当初加入 MiniMax 的另一个理由——这一段其实是整篇最朴实的话:

“我当时用了 ChatGPT,它帮我写一些代码之后(那时代码可能没有那么强)——我就意识到了一点:打不过,就加入。 就相当于你有了蒸汽机、电力之后你不用,人家用热兵器你用冷兵器,那你肯定要被怼死。”

他用一句话给整场收尾:

“新版本来了——还是要看一下新版本的玩法。 积极乐观地拥抱新版本——相信新版本不是滔天大浪把我们抹掉,而是它会成为让我们可以在里面自由翱翔的舞台。”

Koji 也感叹:”我们今天没想到是如此乐观的一个收尾。”——谢谢科技、谢谢 B 站、谢谢大家陪伴。

几条朴素的实战收束

如果把整场两小时的对谈浓缩成几条关于”Agent Harness 时代如何驾驭 AI”的朴素判断,大概是这样:

  • 第一道是承认人是瓶颈:模型能跑得越久越 reliable,人就越成为效率瓶颈;自我进化是这个时代里最有杠杆的设计,而不是抖音算法那种被动学习。
  • 第二道是 multi-agent 不是 nice-to-have:单 agent 上下文一过半就指数降质,两个 agent 互相 closeloop check 是更便宜的”换脑子”方式。
  • 第三道是 skill + CLI 比 MCP 更普惠:让普通人能写、能分享,scale 才能跑出来——脱离 Claude Hub 这种社区基础设施,OpenCode 也不会火。
  • 第四道是 everything is coding:做通用办公、finance、法律,最终都是把”软件”这一层重做——你每天用 Excel,本来就是在写软件。
  • 第五道是对模型公司的差距判断要诚实:训练方法没差太多,真正差距在”任务定义”——也就是请最优秀的人,蒸馏他们的思考变成训练数据。
  • 第六道是对 agent 创业的祛魅:identity 和 payment 不是创业公司能 handle 的;往上一层有机会,但要有领域 know-how;sandbox/memory infra 这类没那么本质。
  • 第七道是不要押泡沫期的姿势:agent 这一层有生命周期——一代版本一代神——Manus 定义的范式已经被新一代收敛掉;外面那一层在越来越薄。
  • 第八道是一句很朴素的话:打不过,就加入。

最后留一句阿岛在聊”花叔蒸馏 skill”那段顺口说的话,我觉得挺适合放在最后——

“最终还是为了让人类可以去真正做自己喜爱的事情——那些有创造力的部分。 那些繁重、重复的部分,就像我们过去发明蒸汽机、发明电力一样。”
