Charles 抓包实战：HTTP 与 HTTPS 调试的必备利器

日期： 2019年05月27日

在日常开发和联调中，网络请求的调试是绑定不了的环节。无论是排查接口返回异常、验证请求参数，还是分析第三方 API 的交互细节，一个好用的抓包工具都能大幅提升效率。Charles 就是这样一款广受开发者青睐的 HTTP/HTTPS 代理调试工具。

什么是 Charles

Charles 是一款跨平台的 HTTP 代理服务器（HTTP Proxy）/ HTTP 监视器（HTTP Monitor）/ 反向代理（Reverse Proxy），它允许开发者查看所有通过代理的 HTTP 和 SSL/HTTPS 流量。这意味着你可以查看请求、响应以及 HTTP 头（包含 cookies 和缓存信息）。

官方下载地址：https://www.charlesproxy.com/

Charles 的核心功能

1. 请求/响应查看

Charles 能够以结构化的方式展示 HTTP 请求和响应的完整信息，包括：

  • Request Headers：查看请求头中的 Content-Type、Authorization、Cookie 等关键信息
  • Request Body：表单数据、JSON 请求体等一目了然
  • Response Headers：服务端返回的状态码、Cache-Control、Set-Cookie 等
  • Response Body：支持 JSON、XML、HTML 等多种格式的格式化展示

2. 断点调试（Breakpoints）

Charles 允许在请求发出前或响应返回前设置断点，你可以：

  • 修改请求参数后再发送到服务器
  • 修改服务器返回的数据后再传递给客户端
  • 模拟各种异常响应，测试客户端的容错能力

3. Map Local / Map Remote

  • Map Local：将某个远程 URL 映射到本地文件，适用于前端资源的本地调试
  • Map Remote：将请求转发到另一个 URL，常用于接口环境切换

4. 限速模拟（Throttling）

模拟慢速网络环境（如 3G、2G），测试应用在弱网下的表现，这对移动端开发尤为重要。

5. 重发请求（Repeat / Repeat Advanced）

可以快速重发某个请求，也可以设置并发数和重复次数进行简单的压力测试。

HTTPS 抓取配置

HTTPS 抓包是 Charles 使用中最常见也最容易踩坑的环节。由于 HTTPS 的 SSL/TLS 加密机制，Charles 需要作为”中间人”来解密流量。核心步骤如下：

PC 端配置

  1. 安装 Charles 根证书：菜单栏选择 Help → SSL Proxying → Install Charles Root Certificate，将证书安装到系统信任的根证书存储中
  2. 开启 SSL Proxying：进入 Proxy → SSL Proxying Settings，添加需要抓取的域名（可使用 * 通配符匹配所有域名）
  3. 验证：访问 HTTPS 站点，确认 Charles 中能看到解密后的明文内容

移动端配置

  1. 确保手机和电脑在同一局域网
  2. 手机设置 HTTP 代理，指向电脑的 IP 地址和 Charles 的代理端口（默认 8888）
  3. 手机浏览器访问 chls.pro/ssl 下载并安装证书
  4. iOS 注意：安装后还需在 设置 → 通用 → 关于本机 → 证书信任设置 中手动开启信任
  5. Android 7.0+：应用默认不再信任用户安装的 CA 证书，需要在应用的 network_security_config.xml 中配置或使用 root 设备

常见使用场景

场景 具体做法
接口联调 查看请求参数和返回值是否符合预期
Mock 数据 使用 Map Local 返回本地 JSON 文件
环境切换 使用 Map Remote 将线上请求重定向到测试环境
弱网测试 使用 Throttling 模拟 3G/丢包
性能分析 查看接口耗时，定位慢请求
排查缓存问题 检查 Cache-Control、ETag 等缓存头

使用技巧

  1. 善用过滤：在 Filter 栏输入域名关键词，过滤掉无关请求，专注于目标接口
  2. 保存 Session：可以将整个抓包会话保存为 .chls 文件，便于与同事共享排查
  3. Focus 功能：右键某个域名选择 Focus，可以将其独立展示，减少干扰
  4. 对比请求：选中两个请求后右键选择 Compare，快速对比差异

替代工具对比

  • Fiddler：Windows 平台免费，功能与 Charles 类似，但 Mac 上体验较差
  • mitmproxy：命令行工具，适合自动化测试和 CI/CD 场景，支持 Python 脚本扩展
  • Wireshark：网络协议层面的抓包，比 Charles 更底层，适合排查 TCP/UDP 层问题
  • Chrome DevTools：浏览器内置，适合 Web 开发，但无法捕获非浏览器的流量

注意事项

  • Charles 本质是中间人代理，不要在生产环境中使用，避免敏感数据泄露
  • 抓包完成后记得关闭系统代理设置，否则 Charles 关闭后网络将不可用
  • 部分应用做了 SSL Pinning（证书锁定），即使安装了 Charles 证书也无法抓取，需要配合 Frida 等工具绕过

总结

Charles 是前后端开发者、测试人员和移动端工程师日常工作中不可或缺的调试工具。掌握它的核心功能——请求查看、HTTPS 解密、断点修改和 Map 映射，能让你在接口联调和问题排查中事半功倍。如果你还没有尝试过，不妨现在就下载体验。
