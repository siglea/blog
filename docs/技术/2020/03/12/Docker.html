## --name nginx-test：容器名称。
## -p 8080:80： 端口进行映射，将本地 8080 端口映射到容器内部的 80 端口。
## -d nginx： 设置容器在在后台一直运行。

查看运行的容器

docker ps
docker ps -a
# 输出详情介绍：
## CONTAINER ID: 容器 ID。
## IMAGE: 使用的镜像。
## COMMAND: 启动容器时运行的命令。
## CREATED: 容器的创建时间。

## STATUS: 容器状态。
## 状态有7种：
### created（已创建）
### restarting（重启中）
### running（运行中）
### removing（迁移中）
### paused（暂停）
### exited（停止）
### dead（死亡）

## PORTS: 容器的端口信息和使用的连接类型（tcp\udp）。
## NAMES: 自动分配的容器名称。

docker中run和start的区别

  • docker run 后面指定的是一个镜像
  • 而docker start指定的是一个容器
  • docker run是利用镜像生成容器，并启动容器，而docker start是启动一个之前生成过的容器
