数据结构是程序设计的基石。Niklaus Wirth 的经典公式"程序 = 数据结构 + 算法"揭示了数据结构在软件开发中的核心地位。选择合适的数据结构不仅能让代码更优雅，更能将算法的时间复杂度从 O(n²) 降低到 O(log n) 甚至 O(1)。

本文将系统梳理最核心的数据结构——从最基础的数组和链表，到进阶的跳表、树、堆和图，帮你建立完整的数据结构知识体系。

---

## 数组（Array）

数组可以说是最基本最常见的数据结构。数组一般用来存储相同类型的数据，可通过数组名和下标进行数据的访问和更新。数组中元素的存储是按照先后顺序进行的，同时在内存中也是按照这个顺序进行**连续存放**。数组相邻元素之间的内存地址间隔一般就是数组数据类型的大小。

**核心特点**：

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 随机访问 | O(1) | 通过下标直接计算内存地址 |
| 尾部插入 | O(1) | 不需要移动元素 |
| 中间插入 | O(n) | 需要移动后续元素 |
| 中间删除 | O(n) | 需要移动后续元素 |
| 查找（无序） | O(n) | 需要遍历 |
| 查找（有序） | O(log n) | 可以使用二分查找 |

数组的最大优势是**随机访问的 O(1) 时间复杂度**和良好的**缓存局部性**（CPU 缓存行会预加载相邻元素）。当你需要频繁按下标访问数据，且插入删除不频繁时，数组是最优选择。

---

## 链表（Linked List）

链表相较于数组，除了数据域还增加了**指针域**用于构建链式存储数据。链表中每一个节点都包含此节点的数据和指向下一节点地址的指针。由于是通过指针进行下一个数据元素的查找和访问，使得链表的自由度更高。

这表现在对节点进行增加和删除时，只需要对上一节点的指针地址进行修改，而无需变动其它节点。不过指针带来高自由度的同时，自然会牺牲数据查找的效率和多余空间的使用。

**常见类型**：
- **单链表**：最基本的形式，每个节点只有一个 next 指针
- **双向链表**：每个节点同时有 prev 和 next 指针，支持双向遍历
- **循环链表**：尾节点的 next 指向头节点，形成环形结构

**核心特点**：

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 随机访问 | O(n) | 必须从头遍历 |
| 头部插入 | O(1) | 只需修改指针 |
| 中间插入 | O(1)* | 已知位置时只需修改指针 |
| 删除 | O(1)* | 已知位置时只需修改指针 |
| 查找 | O(n) | 需要遍历 |

> *前提是已经定位到了目标节点，定位本身需要 O(n)

当你的场景需要频繁在头部或中间进行插入删除操作，且不需要频繁随机访问时，链表优于数组。Java 中的 `LinkedList`、操作系统的进程调度队列都是链表的典型应用。

---

## 跳表（Skip List）

链表虽然通过增加指针域提升了自由度，但数据查询效率低下。特别是当链表很长时，查找某个元素只能从头依次遍历，时间复杂度为 O(n)。跳表的产生就是为了解决这个问题。

跳表通过增加链表的**多级索引**来加快原始链表的查询效率。其基本思想是：在原始链表之上建立多层索引链表，每一层是下一层的"快速通道"。查询时先在高层索引中快速定位大致范围，再逐层下降到原始链表中精确定位。

这样的方式可以让查询的时间复杂度从 O(n) 提升至 **O(log n)**，插入和删除的效率也能达到 O(log n)，与红黑树和平衡二叉树不相上下。

**实际应用**：
- **Redis** 的有序集合（Sorted Set）底层实现
- **LevelDB** 的 MemTable 实现

跳表相比红黑树的优势在于实现简单、支持范围查询、并发友好（分段加锁更容易实现）。

---

## 栈（Stack）

栈是一种**后进先出（LIFO，Last In First Out）**的数据结构。只能在栈顶进行入栈（push）和出栈（pop）操作。

**典型应用**：
- 函数调用栈：每次函数调用都会将返回地址和局部变量压栈
- 表达式求值：中缀表达式转后缀表达式
- 括号匹配：编译器的语法检查
- 浏览器的后退功能
- 撤销操作（Ctrl+Z）

---

## 队列（Queue）

队列是一种**先进先出（FIFO，First In First Out）**的数据结构。在队尾入队（enqueue），在队头出队（dequeue）。

**常见变体**：
- **双端队列（Deque）**：两端都可以入队和出队
- **优先队列（Priority Queue）**：按优先级出队，底层通常用堆实现
- **循环队列**：用数组实现的环形队列，避免出队后的空间浪费
- **阻塞队列**：线程安全的队列，满时入队阻塞，空时出队阻塞

**典型应用**：
- 消息队列（MQ）的核心模型
- 线程池的任务调度
- BFS（广度优先搜索）
- 操作系统的进程调度

---

## 树（Tree）

别看树好像很高级，其实可以看作是**链表的高配版**。树的实现就是对链表的指针域进行了扩充，增加了多个地址指向子结点。同时将"链表"竖起来，从而凸显了结点之间的层次关系，更便于分析和理解。

### 树的遍历方式

树的遍历是最基础的操作，四种遍历方式必须熟练掌握：

- **前序遍历**：根结点 → 左子树 → 右子树
- **中序遍历**：左子树 → 根结点 → 右子树（对 BST 来说即为有序遍历）
- **后序遍历**：左子树 → 右子树 → 根结点
- **层次遍历**：按层从上到下、从左到右遍历（通常用队列实现）

## 平衡二叉树（AVL Tree）

平衡二叉树又称 AVL 树，它是一棵二叉排序树，且具有以下性质：它是一棵空树或它的**左右两个子树的高度差的绝对值不超过 1**，并且左右两个子树都是一棵平衡二叉树。

AVL 树通过旋转操作（左旋、右旋、左右双旋、右左双旋）来维持平衡。虽然查询效率很高，但频繁的旋转使得插入和删除操作的常数因子较大。

---

## 红黑树（Red-Black Tree）

红黑树是一种"近似平衡"的二叉搜索树，相比 AVL 树放宽了平衡条件，通过颜色标记和旋转来维持"黑色平衡"。它的五条性质确保了从根到叶子的最长路径不超过最短路径的两倍。

- 每个结点要么是红的要么是黑的
- 根结点是黑的
- 每个叶结点（NIL 指针或 NULL 结点）都是黑的
- 如果一个结点是红的，那么它的两个儿子都是黑的
- 对于任意结点，其到叶结点 NIL 的每条路径都包含相同数目的黑结点

**实际应用**：Java 的 `TreeMap`、`TreeSet`；Java 8 中 `HashMap` 链表长度超过 8 时转为红黑树；Linux 内核中 epoll/select 的句柄集管理。

---

## 堆（Heap）

堆通常是一个可以被看做一棵树的**数组对象**。堆的具体实现一般不通过指针域，而是通过构建一维数组与二叉树的父子结点进行对应，因此堆总是一棵**完全二叉树**。

**核心性质**：
- **最大堆（大根堆）**：堆中某个节点的值总是不大于其父节点的值，根节点最大
- **最小堆（小根堆）**：堆中某个节点的值总是不小于其父节点的值，根节点最小

**典型应用**：
- 堆排序：时间复杂度 O(n log n)，空间复杂度 O(1)
- 优先队列的底层实现
- Top-K 问题：用大小为 K 的最小堆维护最大的 K 个元素
- 定时器的实现：用最小堆管理超时事件

---

## 散列表（Hash Table）

散列表通过某种哈希算法将键（Key）映射到数组的一个位置（桶），从而实现 O(1) 的查找、插入和删除。

**核心挑战是哈希冲突**——不同的 Key 可能被映射到相同的桶。常见的冲突解决方案包括：
- **链地址法**：每个桶维护一个链表（Java 的 `HashMap` 采用此方案，链表过长时转为红黑树）
- **开放地址法**：冲突时探测下一个空位（线性探测、二次探测、双重哈希）

**典型应用**：字典、缓存系统、数据库索引（Hash Index）、布隆过滤器等。

---

## 图（Graph）

图是一种多维数据存储结构，实际应用中通过图这种模式建立索引与关联关系。图由顶点（Vertex）和边（Edge）组成，能够表达比树更复杂的关系。

### 图数据库

图数据库（Graph Database）并非存储图片的数据库，而是以图这种数据结构存储和查询数据。

- 图形数据库是一种在线数据库管理系统，具有处理图形数据模型的 CRUD 操作
- 与其他数据库不同，**关系在图数据库中占首要地位**，不需要使用外键或 MapReduce 来推断数据连接
- 数据模型更加简单、更具表现力
- 为 OLTP 系统设计，考虑了事务完整性和操作可用性

**典型应用**：社交网络关系图谱、知识图谱、推荐系统、欺诈检测等。主流的图数据库包括 Neo4j、JanusGraph、Amazon Neptune 等。

## 数据结构选型指南

| 需求场景 | 推荐数据结构 | 原因 |
|----------|-------------|------|
| 频繁随机访问 | 数组 | O(1) 下标访问 |
| 频繁头部插入/删除 | 链表 | O(1) 指针修改 |
| 有序数据快速查找 | 跳表/红黑树 | O(log n) 查找 |
| 后进先出 | 栈 | 天然 LIFO |
| 先进先出 | 队列 | 天然 FIFO |
| 快速查找最大/最小值 | 堆 | O(1) 访问堆顶 |
| 键值快速查找 | 散列表 | O(1) 哈希映射 |
| 层次关系 | 树 | 天然层次结构 |
| 复杂关联关系 | 图 | 表达任意关系 |

---

## 总结

数据结构是一切算法和系统设计的基础。从线性结构（数组、链表、栈、队列）到非线性结构（树、图），从朴素实现到优化变体（跳表、红黑树、B+ 树），每种数据结构都有其最适合的应用场景。理解它们的内部原理和性能特征，才能在工程实践中做出正确的选型决策。
